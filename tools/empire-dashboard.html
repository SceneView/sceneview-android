🎯 User Journey — De la découverte au paiement

Funnel basé sur les données npm + gateway. Le point de conversion est le passage Free → Pro quand l'utilisateur dépasse les limites du tier gratuit (rate limiting, tools restreints).

Triggers de conversion

  • Rate limit atteint (10 req/min free) Implémenté
  • Outils Pro bloqués (analyze_project) Implémenté
  • CTA sponsor (chaque 10e appel) Implémenté

Parcours type

1. Dev cherche "android 3d" → trouve SceneView
2. Installe le MCP : npx sceneview-mcp
3. Utilise les tools free (create_scene, etc.)
4. Atteint le rate limit ou veut analyze_project
5. Voit le CTA → checkout Stripe → API key
6. Passe en mode hosted (Pro €19/mo)

🔍 Ce que la telemetry révèlera

Quels tools sont utilisés ?

create_scene, load_model, analyze_project... Savoir quels tools sont populaires aide à prioriser le développement et à comprendre les use cases réels.

Qui dépasse les limites ?

Les users qui atteignent le rate limit sont tes meilleurs prospects Pro. La telemetry te montrera combien et à quelle fréquence — ta source #1 de conversion.

Quelles versions sont actives ?

Savoir si les users sont sur 3.6.x, 4.0.0-beta, ou rc.1 te dit quand tu peux deprecier les anciennes versions sans casser personne.

Quel est le taux d'erreur ?

Les erreurs fréquentes indiquent des problèmes d'API ou de documentation. Chaque erreur est un user potentiellement perdu.
