# SceneView v4.0 — Quality & Release Plan

## Quality First: Zero Tolerance Policy

### Problem
Too many bugs slip through: compilation errors, runtime crashes, broken previews,
CDN issues, store rejections. The user should NEVER find bugs — they should be caught
before push.

---

## Phase 1: Quality Gates (enforce BEFORE every push)

### 1. Pre-push compilation check (mandatory)
```bash
# Must pass before ANY push to main
./gradlew :sceneview:compileReleaseKotlin
./gradlew :arsceneview:compileReleaseKotlin
./gradlew :samples:android-demo:compileDebugKotlin
./gradlew :sceneview-core:allTests
./gradlew :sceneview:test
./gradlew :arsceneview:test
```

### 2. Pre-push website check
```bash
# Validate all HTML files have no broken internal links
# Validate JS files are syntactically correct
node -c website-static/js/sceneview.js
# Validate all referenced assets exist
```

### 3. Bundle build check (for store-affecting changes)
```bash
./gradlew :samples:android-demo:bundleRelease
```

### 4. iOS build check (for Swift changes)
```bash
xcodebuild -scheme SceneViewSwift -destination 'generic/platform=iOS' build
```

## Phase 4: v4.0 Release

### Breaking changes
1. Merge sceneview + arsceneview → sceneview-android
   - ARCore becomes compileOnly (optional)
   - Single dependency: `implementation("io.github.sceneview:sceneview-android:4.0.0")`
   - ARSceneView still works if ARCore is on classpath

2. Remove all @Deprecated from v3.x
   - Scene() → removed (use SceneView)
   - ARScene() → removed (use ARSceneView)
   - raycast/raycastAll → removed (use hitTest)
   - Consumer<Collider> overload → removed

3. Simplify SceneView composable signature
   - Use builder pattern or defaults for most parameters
   - SceneView(modifier) { ModelNode(...) } should work with zero config

### Migration guide
- Dependency change: 2 lines → 1 line
- Import changes: same package, just remove ar prefix for some
- API changes: search-replace table
