# Community

SceneView is built in the open. Whether you have a question, want to share what you built,
or want to contribute — there is a place for you.

---

## Discord

Our Discord server is the best place for real-time help, showcasing your projects, and
chatting with other SceneView developers.

[:fontawesome-brands-discord: Join the SceneView Discord](https://discord.gg/UbNDDBTNqb){ .md-button .md-button--primary }

**Channels you will find:**

- **#general** — Introductions, announcements, and general discussion
- **#help** — Ask questions about SceneView (Android, iOS, Web, Flutter, React Native)
- **#showcase** — Share screenshots and videos of what you built
- **#contributing** — Coordinate contributions and discuss PRs

---

## GitHub Discussions

For longer-form questions, feature proposals, and architecture discussions, use
GitHub Discussions. Threads are searchable and indexed by search engines, so your
question may help someone else in the future.

[:fontawesome-brands-github: Open a Discussion](https://github.com/sceneview/sceneview/discussions){ .md-button }

---

## Stack Overflow

Ask and answer questions using the **`sceneview`** tag. This is the best option when
you want a question-and-answer format that benefits the broader developer community.

[:material-stack-overflow: Browse sceneview on Stack Overflow](https://stackoverflow.com/questions/tagged/sceneview){ .md-button }

---

## Report a Bug or Request a Feature

Found a bug or have an idea? Open an issue on GitHub. Please search existing issues
first to avoid duplicates.

[:octicons-issue-opened-16: Open an Issue](https://github.com/sceneview/sceneview/issues/new){ .md-button }

---

## Contributing

We welcome contributions of all kinds — code, documentation, samples, and bug reports.
See the [Contributing Guide](contributing.md) for how to get started, including the
AI-assisted workflow with Claude Code.
